# Vault agent rules

1. Read `MEMORY.md` at session start.
2. Do not read all files indiscriminately.
3. Prefer `status: active`; normally ignore deprecated and archived entries.
4. Check `updatedAt`, `lastReviewedAt`, provenance, duplicates, conflicts, and supersession.
5. Put AI-created memories in `_staging/`; never write them directly to formal areas.
6. Never activate philosophy or infer profile/preferences from one statement without approval.
7. Prefer archive over deletion and rebuild indexes after approved changes.
8. Never store secrets.
